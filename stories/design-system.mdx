# Design System

This page documents the base visual system that powers the component library.

## Principles

- Calm and airy surfaces
- Soft contrast with clear hierarchy
- Rounded controls with gentle glow on interaction
- System tokens first, component tokens second

## Color Tokens

The raw color tokens are generic and reusable across components. Semantic tokens such as `actionSolidBackground` are composed from them.

## Semantic Token Usage

Use raw tokens for foundations and semantic tokens for component intent.

```ts
colors: {
  brandPrimary: { value: "#47b4eb" },
}

semanticTokens: {
  colors: {
    actionSolidBackground: { value: "{colors.brandPrimary}" },
  },
}
```

This keeps the theme reusable while allowing components to reference intent-based aliases.

## Shared Field Semantics

Helper messaging now follows a shared semantic scale across field components and the standalone `HelperText` primitive:

- `default`: neutral guidance
- `success`: positive confirmation
- `error`: blocking or invalid state
- `warning`: cautionary guidance

This keeps `Input`, `Select`, `Textarea`, and `HelperText` aligned on the same tone language.

## Motion

Micro-interactions should stay soft and readable. Current components favor short entrance motion, subtle glow, and lightweight easing instead of spring-heavy or bouncy behavior.

- Tooltip content uses a short fade-and-rise entrance animation
- Focus feedback relies on glow and border emphasis rather than large scale transforms
